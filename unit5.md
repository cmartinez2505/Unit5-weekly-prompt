I chose Chapter 11 question “Explain how testing code improves your written code”

Writing and running tests in Python helps with troubleshooting bugs and improves the way a developer designs the code. When one knows when to test a function or a class, they keep it simple. For example, if the developer makes the function more complicated, like making it too long or programming it to have a lot of responsibilities at once, writing the test for it can be really confusing and stressful. I found out that to avoid this as much as possible, just break the code down into smaller and easier pieces. After doing this, the programmer is now less stressed and confused because they can now see which function and class do what.


Another way testing improves code is that it can make you think of possible mistakes before you finish the specific part you are working on. For example, if you are writing an input code, you can consider invalid input types like an integer, a string, or even blank fields. So, the key to this topic is to plan for issues before coding a feature and to be prepared before the issues strike the code.
